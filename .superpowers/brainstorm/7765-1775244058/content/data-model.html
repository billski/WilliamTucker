Data Model — Core Entities

The foundation of the business admin system. Review the entities and relationships.

Entity Relationship Overview
CONTACT (Lead/Client)
id, name, email, company
phone, source (calendly/referral/other)
status: lead → prospect → client → archived
notes, created_at, updated_at
PROPOSAL
id, contact_id, title
service_type (assessment/implementation/retainer/workshop)
billing_model (fixed/retainer/day_rate)
amount, description, valid_until
status: draft → sent → accepted / declined
PROJECT
id, contact_id, proposal_id
name, description, billing_model
rate (hourly/daily/fixed/monthly)
budget, start_date, end_date
status: planned → active → completed → archived
TIME ENTRY
id, project_id, date
hours, description
billable (bool) — optional for fixed/retainer
invoice_id (null until invoiced)
INVOICE
id, project_id, contact_id
number (WTS-2026-001), issue_date, due_date
subtotal, tax_rate, tax_amount, total
line_items (JSON array)
status: draft → sent → paid / overdue
payment_method, payment_date, notes
EXPENSE
id, project_id (optional)
category (software/travel/office/marketing/other)
amount, date, vendor, description
receipt_url (Supabase Storage)
tax_deductible (bool)
ACTIVITY LOG
id, contact_id, project_id
type (call/email/meeting/note/follow_up)
description, date, follow_up_date
Tracks all interactions — CRM history
DOCUMENT
id, project_id, contact_id
type (proposal_pdf/contract/deliverable/receipt)
name, file_url (Supabase Storage)
visible_to_client (bool)
Controls what shows in client portal
Key Relationships:
Contact → has many Proposals, Projects, Invoices, Activities
Proposal → accepted → creates Project
Project → has many Time Entries, Invoices, Documents, Expenses
Invoice → links to Time Entries (for hourly billing)
All tables → RLS policies scoped by user_id (admin) or contact_id (client portal)
