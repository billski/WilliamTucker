Admin vs Client Portal — Feature Comparison

What each role can see and do

Feature Admin (You) Client Portal
Dashboard ✓ Full KPIs, all clients ✓ Their overview only
Contacts / CRM ✓ Full CRM, all contacts — Not visible
Proposals ✓ Create, send, track all ✓ View & accept their proposals
Projects ✓ Full management, all projects ✓ View status, description, dates
Time Tracking ✓ Log & manage all entries — Not visible

Expenses ✓ Full expense tracking — Not visible
Documents ✓ All documents, toggle visibility ✓ Only docs marked visible

Authentication & Security:
Admin: You log in with email/password via Supabase Auth. Single admin account.
Clients: Magic link (passwordless email login). You create their account when they become a client. No password to manage.
Data isolation: Supabase Row-Level Security ensures clients can ONLY access rows where contact_id matches their auth user. Enforced at the database level — no app-level bugs can leak data.
Future-proof: If you hire staff, add a role column to the users table. RLS policies already support it.
