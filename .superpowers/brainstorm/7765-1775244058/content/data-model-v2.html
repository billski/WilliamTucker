Data Model — Core Entities

William Tucker Solutions — business admin system foundation

CONTACT (Client Profile & CRM)
Identity
name, email, phone
company, job_title
website, linkedin_url
avatar_url
Business Details
industry, company_size
address (street, city, province, postal)
billing_email (if different)
preferred_payment_method
CRM & Pipeline
source (calendly / referral / networking / website / other)
referred_by (self-ref contact_id)
tags (array — e.g. "finance", "AI-ready")
lifetime_value (computed)
Status Flow
lead → prospect → client → archived
Lead = discovery call booked/initial contact  |  Prospect = proposal sent  |  Client = signed engagement  |  Archived = inactive
PROPOSAL
contact_id, title
service_type (assessment / implementation / retainer / workshop)
billing_model (fixed / retainer / day_rate)
amount, description, scope_of_work
valid_until, terms_and_conditions
draft → sent → accepted / declined
PROJECT
contact_id, proposal_id
name, description, billing_model
rate (hourly / daily / fixed / monthly)
budget, start_date, end_date
deliverables (JSON array)
planned → active → completed → archived
TIME ENTRY
project_id, date, hours
description
billable (bool) — optional for fixed/retainer
invoice_id (null until invoiced)
INVOICE
project_id, contact_id
number (WTS-2026-001)
issue_date, due_date
subtotal, tax_rate, tax_amount, total
line_items (JSON array)
payment_method, payment_date, notes
draft → sent → paid / overdue
EXPENSE
project_id (optional)
category (software / travel / office / marketing / other)
amount, date, vendor, description
receipt_url (Supabase Storage)
tax_deductible (bool)
ACTIVITY LOG
contact_id, project_id
type (call / email / meeting / note / follow_up)
description, date
follow_up_date
Full interaction history per contact
DOCUMENT
project_id, contact_id
type (proposal_pdf / contract / deliverable / receipt)
name, file_url (Supabase Storage)
visible_to_client (bool)
Controls client portal visibility
Key Relationships:
Contact → has many Proposals, Projects, Invoices, Activities, Documents
Proposal → accepted → creates Project
Project → has many Time Entries, Invoices, Documents, Expenses
Invoice → links to Time Entries (for hourly billing)
Contact.referred_by → self-referencing for referral tracking
All tables → RLS: admin sees all, client portal scoped by contact_id + visible_to_client
